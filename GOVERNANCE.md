# Project Governance

This document describes the governance model for ArchiGram.ai.

## Overview

ArchiGram.ai is an open source project maintained by [Timur Isachenko](https://github.com/isatimur) with help from the community. Our goal is to build the best AI-powered diagramming tool while maintaining a welcoming, inclusive environment for all contributors.

## Roles and Responsibilities

### Project Lead

**Current Lead**: [@isatimur](https://github.com/isatimur)

The Project Lead is responsible for:

- Setting the overall direction and vision for the project
- Making final decisions on major architectural changes
- Managing releases and versioning
- Maintaining the security of the project
- Resolving conflicts within the community

### Maintainers

Maintainers are trusted contributors who have demonstrated:

- Consistent, high-quality contributions
- Understanding of the codebase and architecture
- Commitment to the project's goals
- Positive interactions with the community

Maintainer responsibilities:

- Reviewing and merging pull requests
- Triaging issues
- Helping onboard new contributors
- Maintaining code quality standards

### Contributors

Anyone who contributes to the project is a contributor. This includes:

- Code contributions (bug fixes, features, tests)
- Documentation improvements
- Bug reports and feature requests
- Helping others in discussions

## Decision Making

### Day-to-day Decisions

- Minor bug fixes and documentation updates can be merged by any maintainer
- Maintainers should wait 24 hours for feedback on non-trivial changes
- If no objections are raised, the PR can be merged

### Major Decisions

Major decisions (architecture changes, new major features, breaking changes) require:

1. An RFC (Request for Comments) issue or discussion
2. A minimum 1-week comment period
3. Approval from the Project Lead
4. Consensus among active maintainers

### Conflict Resolution

If maintainers cannot reach consensus:

1. A discussion period of at least 48 hours
2. If still unresolved, the Project Lead makes the final decision
3. The decision is documented in the relevant issue/PR

## Becoming a Maintainer

Maintainer nominations can be made by:

- Current maintainers
- The Project Lead

Criteria considered:

- Quality and consistency of contributions
- Engagement with the community
- Understanding of project goals and architecture
- Demonstrated good judgment in reviews

The Project Lead makes the final decision on maintainer appointments.

## Code of Conduct

All participants in the ArchiGram.ai community must follow our [Code of Conduct](CODE_OF_CONDUCT.md).

Violations should be reported to the Project Lead at isatimur.work@gmail.com.

## Changes to Governance

Changes to this governance document require:

1. A pull request with the proposed changes
2. A 1-week comment period
3. Approval from the Project Lead
4. Majority support from active maintainers
